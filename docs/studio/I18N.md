# Internationalization (i18n) — xScanner Studio

Translation system for the frontend application.

---

## Overview

xScanner Studio uses **i18next** with React for internationalization:
- Currently: German (primary), English (secondary)
- Scoped translations with feature overrides
- Type-safe translation keys

---

## File Structure

```
src/
├── locales/
│   ├── de/
│   │   └── translation.json    # German translations
│   └── en/
│       └── translation.json    # English translations
└── lib/
    └── i18n.ts                 # i18n setup & helpers
```

## Translation Files

### Namespace Organization

Translations are organized by **scope** (feature areas):

```json
{
  "common": {
    "toast": { "save": { "title": "...", "description": "..." } },
    "action": { "save": "...", "cancel": "..." },
    "status": { "loading": "...", "error": "..." },
    "validation": { "required": "...", "invalid": "..." },
    "fields": { "serialNumber": "...", "metal": "..." }
  },
  "extraction": {
    "title": "...",
    "stats": { "total": "...", "pending": "..." },
    "actions": { "uploadNew": "...", "browse": "..." }
  },
  "auth": {
    "login": { "title": "...", "subtitle": "..." },
    "logout": "..."
  }
}
```

**Scopes:**
- `common.*` - Generic, reusable translations
- `extraction.*` - Extraction feature
- `auth.*` - Authentication

## Usage

### Basic Translation

```typescript
import { useTranslation } from 'react-i18next'

function MyComponent() {
  const { t } = useTranslation()

  return <h1>{t('extraction.title')}</h1>
}
```

### Scoped Translation (with Override)

```typescript
import { useAppTranslation, I18N_SCOPES } from '@/lib/i18n'

function ExtractionList() {
  const { t } = useAppTranslation(I18N_SCOPES.extraction)

  // Tries 'extraction.toast.save.title' → 'common.toast.save.title'
  const title = t('common.toast.save.title')

  return <div>{title}</div>
}
```

**How override works:**
1. Request: `t('common.toast.save.title')` with scope `extraction`
2. First try: `extraction.toast.save.title`
3. Fallback: `common.toast.save.title`
4. Returns first match

### With Interpolation

```typescript
// Translation JSON
{
  "common": {
    "validation": {
      "required": "{{field}} ist erforderlich"
    }
  }
}

// Component
const message = t('common.validation.required', { field: 'Seriennummer' })
// Result: "Seriennummer ist erforderlich"
```

## Helper Functions

### `useAppTranslation(scope)`

Scoped translation hook with feature override support.

```typescript
import { useAppTranslation, I18N_SCOPES } from '@/lib/i18n'

const { t, i18n } = useAppTranslation(I18N_SCOPES.extraction)
```

### `tWithFallback(t, keys, options)`

Try multiple keys in order, return first match.

```typescript
import { tWithFallback } from '@/lib/i18n'

const text = tWithFallback(t, [
  'extraction.toast.save.title',
  'common.toast.save.title'
])
```

### `tScoped(t, scopes, key, options)`

Try key in multiple scopes.

```typescript
import { tScoped } from '@/lib/i18n'

const text = tScoped(t, ['extraction', 'common'], 'toast.save.title')
// Tries: 'extraction.toast.save.title', 'common.toast.save.title'
```

## Adding New Translations

### 1. Add to Both Languages

**Always add keys to BOTH `de/translation.json` AND `en/translation.json`.**

```json
// de/translation.json
{
  "extraction": {
    "upload": {
      "success": "Upload erfolgreich"
    }
  }
}

// en/translation.json
{
  "extraction": {
    "upload": {
      "success": "Upload successful"
    }
  }
}
```

### 2. Keep Structure Consistent

- Same nesting depth
- Same interpolation variables
- Same key names

### 3. Use Descriptive Keys

```typescript
// ✅ Good
t('extraction.upload.success')
t('common.validation.required')

// ❌ Bad
t('msg1')
t('error')
```

## Scoped Override Pattern

### Purpose

Features can override generic `common.*` translations:

```json
// common.toast.save.title
"Gespeichert"

// extraction.toast.save.title (override)
"Extraction gespeichert"
```

### Usage

```typescript
const { t } = useAppTranslation(I18N_SCOPES.extraction)

// Uses 'extraction.toast.save.title' if exists,
// falls back to 'common.toast.save.title'
t('common.toast.save.title')
```

### When to Override

- Feature-specific wording needed
- More context-specific messages
- Different tone/style

### When NOT to Override

- Generic message works fine
- No added context needed
- Would be redundant

## Best Practices

### DO ✅

1. **Add to both languages**
   ```typescript
   // Always update de/ AND en/
   ```

2. **Use descriptive keys**
   ```typescript
   t('extraction.upload.success')
   ```

3. **Keep leaf values as strings**
   ```json
   { "title": "Text" }  // ✅
   ```

4. **Use interpolation for dynamic values**
   ```typescript
   t('common.validation.required', { field: 'Serial' })
   ```

5. **Organize by feature scope**
   ```json
   {
     "extraction": { ... },
     "auth": { ... }
   }
   ```

### DON'T ❌

1. **Don't use non-string values**
   ```json
   { "count": 5 }  // ❌
   { "enabled": true }  // ❌
   ```

2. **Don't duplicate keys**
   ```json
   // Same text in multiple places - use common.*
   ```

3. **Don't hardcode text**
   ```typescript
   <h1>Willkommen</h1>  // ❌
   <h1>{t('welcome.title')}</h1>  // ✅
   ```

4. **Don't mix languages**
   ```json
   { "title": "Welcome", "subtitle": "Willkommen" }  // ❌
   ```

## Language Switching

Currently German is the default. To add language switching:

```typescript
import { useTranslation } from 'react-i18next'

function LanguageSwitcher() {
  const { i18n } = useTranslation()

  return (
    <select value={i18n.language} onChange={(e) => i18n.changeLanguage(e.target.value)}>
      <option value="de">Deutsch</option>
      <option value="en">English</option>
    </select>
  )
}
```

```typescript
const { t } = useAppTranslation(I18N_SCOPES.upload)
```

## Configuration

i18n is configured in `/src/lib/i18n.ts`:

```typescript
i18n.use(initReactI18next).init({
  resources,
  supportedLngs: ['de', 'en'],
  fallbackLng: 'de',          // Default language
  debug: import.meta.env.DEV, // Log missing keys in dev
  load: 'languageOnly',       // 'de' not 'de-DE'
  interpolation: {
    escapeValue: false        // React escapes by default
  }
})
```

## Troubleshooting

### Missing Translation

Check console in dev mode:
```
i18next::translator: missingKey de translation extraction.invalid.key
```

Fix: Add key to both `de/` and `en/` translation files.

### Wrong Translation

1. Check key path is correct
2. Verify JSON structure matches
3. Check scope is set correctly
4. Clear browser cache

### Interpolation Not Working

Ensure template variables match:
```json
// de/translation.json
"required": "{{field}} ist erforderlich"

// en/translation.json
"required": "{{field}} is required"
```

Usage:
```typescript
t('common.validation.required', { field: 'Name' })
```
